Filter Reordering Options

5 approaches to let users reorder filters in the pipeline.

Option 1: Drag & Drop Chips

Hard

Drag chips to reorder. Most intuitive but requires drag-drop library (dnd-kit or similar).

Format: ⋮⋮ Map answers × ⋮⋮ Suffix: ! × ⋮⋮ Join: ", " ×
Pros: Most intuitive, works on touch
Cons: Needs dnd-kit (~8KB), complex state management
Effort: ~3-4 hours

Option 2: Arrow Buttons on Hover

Easy

Show ← → arrows when hovering a chip. Click to move.

Format: Map answers × ← Suffix: ! → × Join: ", " ×
Pros: No dependencies, simple to implement
Cons: Hover doesn't work on touch, extra UI clutter
Effort: ~30-45 minutes

Option 3: Select + Arrow Keys

Easy

Click chip to select, then use ← → arrow keys to move it.

Format: Map answers × Suffix: ! × Join: ", " ×
Press ← or → to move selected filter
Pros: Keyboard accessible, clean UI
Cons: Not discoverable, needs hint text
Effort: ~45 minutes

Option 4: Position Number Dropdown

Easy

Each chip shows its position. Click number to change position via dropdown.

Format: Map answers × Suffix: ! × Join: ", " ×
Pros: Works everywhere, explicit
Cons: Cluttered, not intuitive
Effort: ~30 minutes

Option 5: Right-Click Context Menu

Medium

Right-click chip for "Move left" / "Move right" options.

Format: Map answers × Suffix: ! × Join: ", " ×
← Move left
→ Move right
Remove
Pros: Clean default UI, discoverable on right-click
Cons: Hidden functionality, no touch support
Effort: ~1-2 hours
Recommendation
Option 2 (Arrow Buttons) is the best balance:
  • Simple to implement (~30 min)
  • No dependencies
  • Clear affordance - users see the arrows
  • Can combine with Option 3 (arrow keys) for accessibility
