Pills With Filters Visible

Filters: default, prepend, join, map

Option 1: Filter names only (no arguments)

Show filter names but hide the argument values.

Patiënt verwezen via Verwijzend ziekenhuis || default || prepend || join.
You see what filters, but not the values

Option 2: Abbreviated filter names

Short codes for each filter.

Patiënt verwezen via Verwijzend ziekenhuis DEF PRE JOIN.
Compact but needs learning

Option 3: Compact pipe chain

Smaller text for filters, no arguments.

Patiënt verwezen via Verwijzend ziekenhuis| default | prepend | join.
Filters are secondary, field name is primary

Option 4: Field → Filters with arrow

Clear separation between field and processing.

Patiënt verwezen via Verwijzend ziekenhuis → default, prepend, join.
Shows data flow direction

Option 5: Mini badges per filter

Each filter as a tiny colored badge.

Patiënt verwezen via Verwijzend ziekenhuis def pre join .
Scannable, each filter distinct

Option 6: Filters as subscript

Filters below the baseline, like a footnote.

Patiënt verwezen via Verwijzend ziekenhuisdefault·prepend·join.
Field prominent, filters subtle

Option 7: Allow wrap (current but no truncate)

Just let long pills wrap to next line.

Patiënt verwezen via Verwijzend ziekenhuis || default: "MM" || prepend: "azeaze" || join: ", ".
Shows everything, takes more space

Option 8: Truncate args only, keep filter names

Show filter names, truncate/hide their arguments.

Patiënt verwezen via Verwijzend ziekenhuis || default || prepend || join.
Best of both - you see what's applied
Recommendation
Option 8 or Option 3 - keep filter names visible but strip arguments. You can see "this field has default, prepend, join" without the clutter of values.
