Generated Expression
The mapping UI generates this filter syntax:
{{ %resource.item.where(linkId='diabetes').answer.valueCoding || map: "yes" => "diabetic", "no" => "non-diabetic", "unknown" => "" }}
Empty string "" = use original display. Unmapped codes also fall back to original display.
Sync Strategy
  • New option added: Show with "NEW" badge, empty mapping field
  • Option removed: Show as "ORPHANED" with option to delete from mapping
  • Option display changed: Show updated display, keep existing mapping
  • Code changed: Treated as remove + add (orphan old, new unmapped)
